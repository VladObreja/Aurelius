## Project Overview

Aurelius is a research and self-tracking protocol knowledge base. It stores a phased biohacking/healing protocol as machine-readable protocol objects, preserves the original research/workbook artifacts, and generates human-readable guides for protocol understanding and execution.

The repository is not a software application. Its architecture is a knowledge-system architecture: canonical data, generated docs, source archive, validation reports, and local BMAD process tooling.

## Getting Started

For protocol understanding:

1. Read [Protocol Spine](./protocol-spine.md).
2. Read [Daily Guide](./daily-guide.md).
3. Use [Protocol Index](./protocol-index.md), [Compound Index](./compound-index.md), and [Practice Index](./practice-index.md) for object lookup.

For BMAD-based protocol development:

1. Start with [BMAD Protocol Workflow](./bmad-protocol-workflow.md).
2. Use this index as BMAD's brownfield project context.
3. Create a Protocol Brief, then Protocol Requirements Document, then Protocol Architecture.

For data work:

1. Review [Data Layout](./data-layout.md).
2. Inspect `data/protocol/*.json`.
3. Run validation after any canonical data changes.

## For AI-Assisted Development

When planning new protocol work, ground the session in:

- `docs/index.md`
- `docs/protocol-spine.md`
- `docs/bmad-protocol-workflow.md`
- `AGENTS.md`

Use BMAD terms in their protocol-adapted sense:

- product -> protocol
- feature -> protocol module/intervention
- implementation -> canonical data and generated docs
- QA -> safety, coherence, evidence, and reference validation
- architecture -> phase model, dependency graph, progression gates, and safety model
