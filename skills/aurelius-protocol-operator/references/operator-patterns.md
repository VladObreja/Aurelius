# Operator Patterns

## Ambiguous Phase

If the user does not specify phase:

- If they ask "where do I start", default to Phase 1.0 / Safe Start.
- If they ask about optimization, ask for current phase and current stability.
- If they mention destabilization, use Safe Mode framing.

## Sunday Morning Pattern

For "It's Saturday evening, where do I start Sunday morning?":

1. Assume Phase 1.0 unless the user says otherwise.
2. Keep Sunday morning small: hydration, binder window, carrot salad, first-meal CORE agents, gentle practices, daily tracking.
3. Defer SUPPORT/EXPERIMENT unless already part of the current phase plan.
4. Emphasize no aggressive training or new experimental additions on day one.

## Shopping Pattern

Group shopping output:

- **Buy now / Phase CORE**
- **Useful but optional / SUPPORT**
- **Do not buy yet / gated**
- **Non-supplement supplies**

Include food/tools where present in protocol data, such as carrot salad ingredients, hydration support, and practice/device requirements when relevant.

## Mild Adverse Signals

Use adverse-event class thinking:

- Energy crash: stop new variables, consider Safe Mode, review recent changes.
- Insomnia/anxiety: review stimulatory, dopaminergic, endocrine, training, and timing load.
- Constipation/bile symptoms: review motility, binders, bile agents, antifungal escalation, hydration.
- Skin/fungal flare: review antifungal/binder/motility context; do not assume worsening is automatically good.
- Libido/EQ crash: review sleep, endocrine/neuro additions, overtraining, vascular practices, and recent changes.

## Protocol Change Requests

If the user asks to add a new intervention:

1. Do not add directly to executable protocol.
2. Create a candidate note or route to BMAD/HIRA.
3. Ask for purpose, phase, entry gate, stop rule, tracking signal, and risk class.
